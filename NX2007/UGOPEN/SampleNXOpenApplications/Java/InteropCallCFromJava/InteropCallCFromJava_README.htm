InteropCallCFromJava

This example demonstrates how you can call existing legacy UF functionality from an NXOpen Java application using JNI. This is one approach to consider if you have extensive investment in old automation programs. This example does not go into detail about the JNI. To find out more about JNI, please reference your platform’s JDK provider's web site (for example, Sun's provided JDK reference is java.sun.com).

Requirements

You will need both the C++ compiler and the Java compiler appropriate for this release.

How to run the example

Unless noted otherwise, the commands in these instructions must be run from an NX command prompt.  See the general notes of the examples for instructions on how to start an NX command prompt.

Instructions for Windows: All commands are the same as in Unix except substitute "make" with "nmake -f Makefile_win".

To build the example

To build the example, you need to build both a C++ DLL and a Java class.

Java class.

To build the Java class and generate the JNI header file, just do a "make". This compiles the Java file and generates the JNI header file using javah.

C++ DLL/SO

You must first build the Java portion before building the C++ portion.  When building the C++ portion, you need to reference the JNI header files in the JDK installation.  We refer to the base directory of the JDK as <JAVA_HOME>.  Additionally, to make things easier, this example DLL/SO is built with the UI libraries.  If it is desired to run this example in batch mode, do not link against the UI libraries.

 

How to Build

 ------------ Windows -------

1. Using Visual Studio, create a new project using the NX Open AppWizard. This is described in the NX Open for C/C++ -> Open C Reference Guide -> Overview -> setting up your system. The image created is an Internal User Function program using C++.  Remove/Delete the default file that is added to the project.

2. Add the UFApp.cpp and JavaApp.h  files by selecting the Project pulldown and selecting "Add Existing Item...". Select the files using the popup window.

3. Now to compile the source, we need to add a reference to the JNI header files in the JDK installation.  We refer to the base directory of the JDK as JAVA_HOME.  This can be done by using the Additional Include Directories property.  As of VS2009, this can be done by using Project->Properties.  Go to the Configuration->C++->General tab.  Add the directory <JAVA_HOME>\include.  Also add the appropriate platform-specific subdirectory in <JAVA_HOME>\include, e.g. for windows 32, this is <JAVA_HOME>\include\win32.

3. Build the external image by going to the Build menu and selecting either "Build Solution" or "Rebuild Solution". This creates a library called File2Points.dll in the Debug folder.

 

The uflink tool can also be used to create the executable by following these steps:

1. Open an NX command prompt window.

2. Change directories to the directory containing the source code.

3. Compile the source file using ufcomp: ufcomp –I<JAVA_HOME>\include –I<JAVA_HOME>\include\<platform directory>, where <JAVA_HOME> is the base directory of the JDK install, and <platform directory> is the platform directory in the include directory, e.g. on Windows 32 this is <JAVA_HOME>\include\win32).

4. Link using uflink: uflink UFDLL=UFapp.dll     This creates UFapp.dll in the current directory.

 

------------ Unix -------

Start up an NX command prompt window, and then start up ufmenu ($UGII_BASE_DIR/ugopen/ufmenu).  For the compile options, modify the options by adding the include directory of the JDK include directory and the machine-dependent include directory, e.g. for Linux this is <JAVA_HOME>/include and <JAVA_HOME>/include/linux).  Link normally using uflink as an internal user function program. Select "i" when prompted for the type of application to link - internal/external. Select "y" for linking with user interface code. Also select "y" for linking a UG/Open++ image.  Keep all the options, and select "y" to link a C++ image. This creates UFApp.so.

Interactive mode
  1. For JNI to work correctly, it needs to load the native library (in our case UFApp).  On Windows this is accomplished by using the PATH environment variable.  On UNIX this is accomplished using the LD_LIBRARY_PATH environment variable.  If you get a stack trace such as the one below when you run the example, this is indicative that the shared library or DLL is not in the path.

        at java.lang.ClassLoader$NativeLibrary.load(Native Method)

        at java.lang.ClassLoader.loadLibrary0(Unknown Source)

        at java.lang.ClassLoader.loadLibrary(Unknown Source)

        at java.lang.Runtime.loadLibrary0(Unknown Source)

        at java.lang.System.loadLibrary(Unknown Source)

        at JavaApp.<clinit>(JavaApp.java:31)

  1. Start NX.

3.   In NX, go to File -> Execute -> NX Open, select JavaApp.class, and press OK.  The results are printed to the syslog.
