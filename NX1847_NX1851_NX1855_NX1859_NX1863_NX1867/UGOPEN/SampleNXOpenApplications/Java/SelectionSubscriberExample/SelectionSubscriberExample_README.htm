SelectionSubscriberExample

This example instantiates a class with an onSelectionChangeCallback method that is registered with the NX SelectionSubscriber. The class with the onSelectionChangeCallback is left loaded until the session terminates. 

Any time the set of selected objects changes due to any user interaction, the SelectionSubscriber will print the current set of deselected tags and selected tags out to the Listing Window. 

The callback may be executed while in Global Selection, or while in a command. It may be executed with or without objects already selected. 

Files required

  1. SelectionSubscriberExample.java

How to run the example

Unless noted otherwise, the commands in these instructions must be run from an NX command prompt.  See the general notes of the examples for instructions on how to start an NX command prompt.

Instructions for Windows: All commands are the same as in Unix except substitute "make" with "nmake -f Makefile_win".

Interactive mode
  1. Compile the example with following command: 
make
  1. Start NX.
  1. Open new part.
  1. In NX, go to File - Execute - NX Open, select SelectionSubscriberExample.jar, and press Ok.  The callback code will be executed as objects are selected and deselected in NX.
