NX™ Open for Java™ examples

How to use these examples

  • Copy the NX Open examples directory to a directory for which you have write-access
  • Set the UGII_LICENSE_FILE and JAVA_HOME environment variables to valid settings.  Start an NX command prompt using the instructions in the section below.  Add the Java Runtime Environment's javac and java executables to your PATH.  Run java -version to verify that java is in your PATH and that you are using the correct version of java.
  • Follow the instructions in the README file for the example that you want to run.
  • If you want to run an example on Unix from an interactive session of NX, start NX through ugmenu or the $UGII_ROOT_DIR/ugii script.

How to start an NX command prompt

On Windows:

Go to Start - Programs.  The shortcut to launch an NX command prompt is located in the same cluster where the shortcut to launch NX is located.

On Unix:

Run ugmenu and select the "UGOPEN-API" option.  Then select "Non-menu activites" and then the shell type.

How to run the examples on Unix

On Unix, some java libraries need to be added to LD_PRELOAD before running a program.  The script $UGII_BASE_DIR/ugopen/run_java should be used in place of the standard java executable to run a program in batch mode.  Use ugmenu or the $UGII_ROOT_DIR/ugii script to start up NX when you want to run a java program inside NX (i.e. interactive mode).

Special settings like these are not required on Windows.

Note for Windows: many of the examples use 'nmake' for compiling; 'nmake' is included with Microsoft Visual Studio.  For those who don't have Visual Studio, 'nmake' can be downloaded free of charge from Microsoft or Lucent.
