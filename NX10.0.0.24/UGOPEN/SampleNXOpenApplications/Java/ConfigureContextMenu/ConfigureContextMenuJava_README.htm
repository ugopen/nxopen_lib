ConfigureContextMenu

This example demonstrates how to customize context menus in NX. It defines basic functionality for making changes to context menu options for both the C Sharp and VB.Net languages.

How to run the example

Unless noted otherwise, the commands in these instructions must be run from an NX command prompt.  See the general notes of the examples for instructions on how to start an NX command prompt.

Instructions for Windows: All commands are the same as in Unix except substitute "make" with "nmake -f Makefile_win".

Batch mode

Not Applicable - the Context Menu is part of the UI

Interactive mode
  1. Compile the example 
    make
  2. Copy the output ConfigureContextMenu.class file into the "startup" directory in one of the following :
    1. $UGII_USER_DIR/startup
    2. For released applications, using UGII_CUSTOM_DIRECTORY_FILE is highly recommended. This variable is set to a full directory path to a file containing a list of root directories for all custom applications. e.g., UGII_CUSTOM_DIRECTORY_FILE=\ugii\menus\custom_dirs.dat
  3. Start NX.
  4. Create a new part
  5. Change the Role to Advanced if not set.
  6. Insert a Block, go to Insert->Design Feature->Block.
  7. Right Click on the Block in the Part Navigator. Notice that Properties has moved to the top of the context menu and is the default option.
  8. Right Click on the Block in the Graphics Window. Notice that here the Delete option has been removed.
  9. Insert a Cylinder, go to Insert->Design Feature->Cylinder.
  10. Right Click on the Cylinder in the Graphics Window. Notice that the submenu "Custom" has been added that includes a Visualization Preferences entry.
