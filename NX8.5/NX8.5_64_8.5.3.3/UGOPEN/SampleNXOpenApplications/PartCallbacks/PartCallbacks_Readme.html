PartCallbacks

Description: This example demonstrates how to register the part callbacks and how the part callbacks are executed.

Files Required for Registering Part Related Callbacks

This table represents the various demos for registering part callback for every available reason via the different languages.
Language Files Note
C++ cpp_part_callbacks.cpp & cpp_part_callbacks.h Both the files must be compiled together
C# cs_part_callbacks.cs You must add a reference to the following .Net component when compiling in Visual Studio: System.Windows.Forms
C ufun_part_callbacks.c
JAVA java_part_callbacks.java
VB vb_part_callbacks.vb

How to Compile the Files: The detailed description on how to compile the files in various languages is available in PartCallbacks_CompileLink_README.html

How to Run the Example

    [A] How to Register the Part Callbacks

    Once you have successfully finished compiling the files, you can follow one of the below methods to register the part callbacks:

    Method 1:
    1. Start NX.
    2. Go to File -> Execute ->NX Open.
    3. You will lead to window called "Execute User Function".
    4. Navigate to the location of the .dll/.so file. Hit ok.

    Method 2:
    1. The output library needs to be placed in a folder called startup (for example c:\user_dir\startup).
    2. Also the UGII_USER_DIR environment variable needs to point to the startup directory's parent folder (for example: UGII_USER_DIR=c:\user_dir).
    3. Once you have the startup directory and user dir environment variable set up, start NX.
    4. The part callbacks will be registered at startup. Now anytime you create new, open, save, save as, close, modify, rename, or change the workpart your part callbacks will be executed.

    [B] How to Execute the Part Callbacks

    Once you have successfully registered the part callbacks, you can execute the callbacks using one of the below methods:

    Method 1:

    Using sample program (execute_all_part_callbacks.c) which works to invoke every possible part related callback.

    1. Create New Part
    2. Save Part As
    3. Modify Part
    4. Save Part
    5. Close Part
    6. Change Workpart (note the workpart changes automatically whenever parts are opened or closed)
    7. Open Part
    8. Rename Part (note rename part happens automatically whenever the part is 'saved as')

    It is used to demo callbacks already registered by one of the following:

    1. cpp_part_callbacks.cpp & cpp_part_callbacks.h
    2. cs_part_callbacks.cs
    3. java_part_callbacks.java
    4. ufun_part_callbacks.c
    5. vb_part_callbacks.vb
    6. or any custom program you write with part related callbacks

    The path and filenames used in this demo are hard-coded. Please validate that the filenames used are valid on your machine or change them as needed before compiling this program.
    The default part names are:

    1. c:\temp\part1_temp.prt
    2. c:\temp\demo_part1.prt

    If you already have part files with those names at those locations on your machine the program will produce errors at runtime. You must delete the parts created at those locations between each time you run this demo program.

    To demo all of the available part callbacks at runtime - compile and link execute_all_part_callbacks.c. Then start NX and go to File->Execute->NX Open and select your compiled execute_all_callbacks program to run the demo.


    Method 2:
    1. This is an interactive method. Once you have registered the Part Callbacks, you can invoke every possible part callback manually.
    2. All the part callbacks can be verified manually by performing the below operations.
      1. Create New Part
      2. Save Part As
      3. Modify Part
      4. Save Part
      5. Close Part
      6. Change Workpart (note the workpart changes automatically whenever parts are opened or closed)
      7. Open Part
      8. Rename Part (note rename part happens automatically whenever the part is 'saved as')
