QuerySketch

This example queries the contents of a sketch.  The example prints the geometry and constraints for the sketch to the NX listing window.  For each geometry, the associated constraints are printed.  For each constraint, the associated geometry is printed.  The example also illustrates how to implement a java OutputStream using the NX listing window.  The example also illustrates interoperability between NX Open for Java and Java UF by using Java UF for some operations and by sharing data between NX Open for Java and Java UF.

How to run the example

Unless noted otherwise, the commands in these instructions must be run from an NX command prompt.  See the general notes of the examples for instructions on how to start an NX command prompt.

Instructions for Windows: All commands are the same as in Unix except substitute "make" with "nmake -f Makefile_win".

Batch mode

make run

The results will be printed to stdout.

Interactive mode
  1. Compile the example
    make
  2. Start NX.
  3. In NX, open a part file that contains a sketch.  If you do not have a part file that contains a sketch, you can create one by running CreateASketch.  To run CreateASketch, go to File - Execute - NX Open, select CreateASketch.class, and press Ok.
  4. Run SketchQueryExample, by going to File - Execute - NX Open.  Then, select SketchQueryExample.class and press Ok.  Select a sketch when prompted by the program.

The results will be printed to the listing window.

Other options

These flags are available for the SketchQueryExample class.

  • -h or -?        Print help
  • -l <logfile>   Write the output to the specified file
