CreateNote

This example creates notes by either copying an existing note or creating a note from a user defined symbol.

How to run the example

Instructions for Windows: All commands are the same as in Unix except substitute "make" with "nmake -f Makefile_win"

  1. Compile the example.
    make
  2. Start NX.
  3. In NX, open a part. If you have one that has a note in it, that would be helpful. If not, open any part or an new/empty part.
  4. Go to File - Execute - NX Open, select CreateNoteExample.jar and press Ok.
  5. On the dialog, select either the "create from user defined symbol" or "copy existing note" option.
    For create from user defined symbol:
    1. Select the name of the user defined symbol and the symbol's size and aspect ratio.
    2. Click the "Create" button.
    3. Pick the location for the new note. The note will be created in the XY plane of the WCS.
      You can create as many copies as you want.
    4. When you are finished, cancel when prompted for the location.

    For copy existing note:
    1. Click the "Create" button.
    2. Select a note to copy. You can select one that existed previously in the part or select
      one that was just created.
    3. Pick the location for the new note. The note will be created in the XY plane of the WCS.
      You can create as many copies as you want.
    4. When you are finished, cancel when prompted for the location.

Design notes

The key portions of code are UserSymbolDialog.getNoteCreationData, UserSymbolDialog.loadSymbolNames, and the call to Annotations.createNote in main(). Most of the rest of the code is just for the GUI for the example. loadSymbolNames() reads the names of the available user defined symbols from the selected Symbol Font Definition file (sbf file). getNoteCreationData() gets the text and preferences for the new note. If "copy existing note" was selected, getNoteCreationData prompts the user to pick a note and gets the text and preferences for the new note from the selected note. If "create from user defined symbol" is selected, the user symbol that was selected is loaded into the part and the user symbol preferences for the new note are constructed based on what the user specified on the form. Then, the main subroutine prompts the user for the location for the new note and creates the note.
